# Design: Add ZeroAlloc.Specification and ZeroAlloc.Results

**Date:** 2026-03-18
**Status:** Approved

## Overview

Add ZeroAlloc.Specification and ZeroAlloc.Results as fully integrated projects in the monorepo — matching the existing pattern established by ZeroAlloc.Validation and ZeroAlloc.Pipeline.

## What Gets Added Per Project

Each project follows the same four-part pattern:

1. **Git submodule** at `repos/<name>` pointing to `https://github.com/ZeroAlloc-Net/ZeroAlloc.<Name>`
2. **Docusaurus app** at `apps/docs-<name>` — reads docs from the submodule, uses shared theme
3. **wrangler.jsonc** in the app — deploys static build to Cloudflare Workers
4. **packages.ts entry** in `apps/web/src/data/packages.ts`

## Docusaurus App Structure

Each app is modelled on `apps/docs-validation`:

```
apps/docs-<name>/
  docusaurus.config.ts      # points to repos/<name>/docs and assets
  package.json              # @zeroalloc/docs-<name>
  sidebars.ts               # autogenerated
  wrangler.jsonc            # assets.directory = ./build
  tsconfig.json
  src/css/custom.css
  static/                   # .nojekyll, favicon fallback
```

**docusaurus.config.ts key values:**
- `staticDirectories: ['static', '../../repos/<name>/assets']`
- `docs.path: '../../repos/<name>/docs'`
- `url: 'https://<name>.zeroalloc.net'`

## packages.ts

Two new entries added after ZeroAlloc.Pipeline, both with `available: true`.

## Cloudflare Deployment

Deploy command per project (set in Cloudflare dashboard):
```
cd apps/docs-<name> && npx wrangler versions upload
```
Build command:
```
pnpm build --filter @zeroalloc/docs-<name>
```
